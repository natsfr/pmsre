incN - Motor increment registers

This defines the value which will be added to the motor posision during each cycle of the command. The value is actually pushed into the command fifo when the len register is written.

Register layout

313029282726252423222120191817161514131211109876543210
unusedsignfrac

Register fields

NameWidthLong nameDescription
frac 24 bits frac Increment value in Q1.23 format, should not be greater than 1.0 (0x800000). This defines the motor speed during the command.
sign 1 bit sign Increment sign bit, defines the motor direction during the next command.

push - Command schedule fifo register (write only)

Writting to this register schedules a command with the specified duration and user id. The increment values of all motors along with the duration and the user id are pushed into the scheduled commands fifo. The command will execute as soon as all previous commands are completed. Execution will be postponed if the fifo of completed commands is full. Both fifos are 4 entries deep.

pop - Completed command fifo register (read only)

This pops a command status from the fifo of completed commands. Up to 4 status can be queued in the fifo.

Register fields

NameWidthLong nameDescription
rem 24 bits rem Number of remaining cycles when the command ended.

flush 1 bit flush This indicates that the command has been flushed due to a fault or stop condition. Any new command will be flushed until the fifo of completed commands becomes empty.
valid 1 bit valid This indicates that the fifo was not empty (other fields are valid).

ctrl - Control register

Enable motor drivers and end-stop inputs watching. Each motor has 2 associated end-stop, one for each direction. Watched end-stops will make any move in the wrong direction flush all commands.
