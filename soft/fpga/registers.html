incN - Motor increment registers

This defines the value which will be added to the motor posision during each cycle of the command. The value is actually pushed into the command fifo when the len register is written.

Register layout

313029282726252423222120191817161514131211109876543210
unusedsignfrac

Register fields

NameWidthLong nameDescription
frac 23 bits frac Increment value in Q1.22 format, should not be greater than 1.0 (0x800000). This defines the motor speed during the command.
sign 1 bit sign Increment sign bit, defines the motor direction during the next command.

len - Command schedule register (write only)

Writting to this register schedule a command with the specified duration and user id. The increment values of all motors along with the duration and the user id are pushed into the scheduled commands fifo. The command will execute as soon as all previous commands get completed. Up to 4 commands can be queued in the fifo.
